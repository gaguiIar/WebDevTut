Introduction to Web Development

By Gerardo Aguilar

Are you looking to break into tech but don't have extensive programming experience? My name is Gerardo, and I'm a sophomore studying Computer Science & Business Administration at the University of Southern California. I believe that with enough dedication, anyone can become a successful developer, so I built this website with that in mind. Introduction to Web Development is a resource aimed to help any beginner looking to branch into tech by breaking down web fundamentals such as HTML, CSS & JavaScript. If you're interested in learning more about web development, I applaud and encourage you to keep reading on. To start, you will want some type of IDE or Integrated Development Environment. I recommend Atom, but Sublime Text or Visual Studio Code are also very popular HTML text editors. You can't go wrong with either, so I recommend you read a bit about all options and pick one you can quickly become familiar with. Don't be afraid to get started; You can use the navigation bar below this paragraph or the directional arrows to navigate the sections of this website.

Hosting

  1. GitHub, Inc. is a provider of Internet hosting for software development and version control. You can sign up for a GitHub account here.

  2. There are two ways to push your code into GitHub from your workstation, the foremost being through a command-line, but the more straightforward way being through GitHub Desktop. I recommend you pick the ladder as it provides a simpler interface, but you can see setup instructions and documentation by clicking either link.

  3. When youâ€™ve set up your preferred method of pushing code, you will want to find out how to make a new repository and extract the contents of the code you want to push to this repository. Once your code is in the same folder of the repository, you can commit to main and then "push" it to GitHub, which will add your code to your online GitHub Repository.

  4. The benefits of pushing your code to your GitHub Repository include being able to keep track of versions of your project, commits, and if you make your repo public contributing to the Git Community as well as free hosting.

  5. Once your code has been committed, you should find your repo on GitHub and find the settings option under the repository name.

  6. If you scroll down to GitHub Pages, you will find a Source option change this to main and save.

  7. Github should now display a URL inside the GitHub pages section, where your project will live. As long as your repository is public, you will be able to share your website through this link for free.

  8. If you would like to use a custom domain, you can follow this guide here.
