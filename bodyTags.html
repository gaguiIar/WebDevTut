Body Tags

As the name implies HTML structure tags establish the foundation of any HTML webpage; However, to actually build the structure of a webpage you’ll want to deal with interal body tags as these are what is actually going to be rendered by your browser.

  1. <div>

    2. The <div> tag creates a division within your body. For example a navigation bar, photo gallery, paragraph, or biography can all be contained within a div container. DIv containers are useful for splitting your complex websites into digestible sections. Div containers can also be used with ids or classes to style specific groups of elements at once. We haven’t covered ID’s and Classes yet so don’t worry if that idea seems a bit complex, it will make more sense later.

  2. <p>

    3. The <p> tag is used to store text and has no special formating besides the fact that a <p> tag will automatically render a blank line before and after every <p> element

  3. <h#>

    4. Header, or <h#> tags can be used to make titles or of course headers within your webpage, Headers by default are bold in text and decrease or increase insize depending on how far the number used in the header tag is from 1, the first, principal heading tag. Possible tag variations are <h1>, <h2>, <h3>, <h4>, <h5>, <h6> with h1 being the largest and h6 being the smallest.

  4. <img>

    5. The img tag is used to add images to your website. For a image to be succesfully rendered you have to use it with the properties src and alt.

    src is used to priovide the location for the source of the image, if you have the image in your directory you can simply lead it to the file but alternatively you can use image urls by utalizing the url(“”) argument. If you will be using multiple photos its best you put them all in one central img file.

    alt is an important accessibility feature. The texts in the alt tag should desribe the image as it is what will be displayed if the image file cannot be found. Additionally for people who use screenreaders alt text will be used to describe what the image is. You should always have accurate and useful alt descriptors.

    Something to note img tags do NOT have a closing image tag.

    This is what a full image tag would look like:

    imgExample.html

    <!DOCTYPE html>

    <html>

    <head>

    <title>Cool Photo</title>

    <link> rel="stylesheet" href="style/master.css">

    </head>

    <body>

    <img src = "imgs/sgm.jpg" alt = "DTLA view">

    </body>

    </html>

    Cool Photo


  5. <a>

    6. The Anchor tag defnies a hyperlink to navigate from one page to another. You can use an anchor tag to direct someone to a already hosted url or a local html file like the img tag.

    Within the opening anchor tag you must utalize the href proeprty to link the website or file the hyperlink should navigate to.

    This is what a full anchor tag would look like:

    imgExample.html

    <body>

    <a href = "index.html"> Home Page <a>

    </body>

  6. <body>

    7. There are all kinds of files you can open on the web and each of them have properties that change the way they are displayed in a brower. Browsers need to be able to identify what kind if file is being loaded to figure out how to display it; the DOCTYPE tag allows us to do so. Writing DOCTYPE declares the document type as an HTML file and tells the browser how it should be displayed.
