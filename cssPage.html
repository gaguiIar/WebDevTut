CSS Grouping with ID's and Classes

display

h1 {

display: block;

}

changes how element is displayed

Other values: none, inline, block, inline-block


width

h1 {

width: 100px;

}

sets width of element

Other values: percentage, inherit, any pixel value or auto


height

h1 {

height: 100px;

}

sets height of element

Other values: percentage, inherit, any pixel value, auto


In the previous page we saw examples using various CSS properties like shown above,

However, as you can see by looking at the CSS Identifier these CSS properties are only being applied to h1 tags. This also means they will be applied to every h1 tag on that HTML file. What if you only wanted to set the hight of ONE h1 tag to 100px? What if you wanted to set the width of h1 tags to 100px but only h1 tags in a certain section of your webpage? This is where IDs and Classes come in handy. Read on to learn more.

  1. IDs & Classes

    If you recall in the internal body tags section we covered div tags which although donâ€™t change the appearance or effect anything visually help us create divisions within our body page. We can consequently use div tags to section out different portions of our website such as a header, title or paragraphs. We can style these new groups by adding id or class selectors to the opening div tags. Doing so will mean any style applied to that specific id or class will effect every element within that div.

    An opening div tag with an id selector looks like so: <div id="test">

    An opening div tag with a class selector looks like so: <div class="test">

  2. When to use IDs over Classes?

    One of the major rules about IDs is they can not appear twice on the same HTML document. This means that ID tags should only be used to style a single element. If you know you will want to reuse some CSS on multiple elements(such as the CSS that makes the example boxes on every page on this website) you should use classes.
