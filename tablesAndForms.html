  2. <form>

    3. Forms allow user interaction and data collection to happen. You start off a form with a <form> tag that contains an action= "" attribute which specifies where form data is sent to upon submission.

    • Input Tags

    Input tags have a type="" attribtue and as the name implies, define what type of input will be taken. Here are some possible type attribtues and other types of input:

    1. type =“text” adds a text-field
    2. type ="email" adds a text-field with basic email validation
    3. type =radio adds radio buttons. You can group radio buttons by setting the same name attribute. Radio buttons are used when you only want one option to be selectable at once.
    4. type ="checkbox" adds a checkbox field, similar to radio buttons except multiple can be selected or “checked” at once.
    • Buttons

    There are two types of buttons you’ll likely use, submit and reset. To add a submit button you’ll start off by inserting <button> tags. Within the opening button tag you’ll want to add a type=“submit” attribute. Between the opening and closing button tags you’ll want to put the text that will be displayed on the button itself

    You can do the same to make a reset button that clears all text-fields/boxes, except replacing the type=“submit” attribute with reset.

    • Larger Input Tags

    When you want a user to select an option from a list you might want to use the drop-down function.

    Drop-down functions work a bit like lists or tables do. You start a drop down function by using a <select> tag. Between the opening and closing select tags goes all your drop down options seperated by an opening and closing <option> tags.

    To add a multi-line textfield instead of using a regular text tag you’ll need to use a textarea input field. You can do this by just incerting <textarea> tags
